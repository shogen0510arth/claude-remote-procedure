ターミナル入門
基本コマンドと操作のコツ

ターミナルは、マウス操作の代わりに「文字でパソコンへ指示する画面」です。最初はここで紹介する操作だけ覚えれば、CodexやClaude Codeを使う際にも十分役立ちます。

対象：初心者 macOS・Linux zsh・bash 一部 Windows PowerShell

目次

    ◎

    ターミナルとは

    ターミナルは、マウス操作の代わりに「文字でパソコンへ指示する画面」です。最初は難しく見えますが、覚える操作はごくわずかです。ここで紹介する内容だけでも、CodexやClaude Codeを使う際に十分役立ちます。

    このページの読み方

    上から順に読めば、初心者が最初に必要な操作をひと通り身につけられる構成です。まずは「最初に覚える基本コマンド」と「操作を速くするコツ」だけでも実用になります。

    1

    画面の見方とプロンプト

    ターミナルを開くと、例えば次のように表示されています。

    画面表示
    taro@MacBook project %

    最後の % や $ の右側にコマンドを入力します。

    画面表示
    taro@MacBook project % pwd

    なお、解説記事の先頭に付いている $ は、通常は入力しません。次のように書かれていても……

    記事の表記
    $ pwd

    実際に入力するのは pwd だけです。

    プロンプトとは

    入力を待っている記号（%・$・# など）を「プロンプト」と呼びます。macOSの標準シェルはzshで、記号は % です。

    2

    最初に覚える基本コマンド

    以下は主にmacOS・Linux用です。Windowsでも、PowerShellなら多くがそのまま使えます。

    コマンド意味使用例
    pwd現在いる場所を確認pwd
    ls現在のフォルダ内を見るls
    cd別のフォルダへ移動cd Documents
    cd ..1つ上のフォルダへ戻るcd ..
    cd ~ホームフォルダへ戻るcd ~
    mkdir新しいフォルダを作るmkdir my-project
    touch空のファイルを作るtouch memo.txt
    catファイル内容を表示cat memo.txt
    cpファイルをコピーcp memo.txt backup.txt
    mv移動・名前変更mv memo.txt note.txt
    rmファイルを削除rm memo.txt
    clear画面を整理clear
    history過去のコマンドを見るhistory
    迷ったら pwd と ls

    ターミナル作業では「いま、どのフォルダにいるのか」が非常に重要です。迷ったら、まず pwd と ls を実行してください。

    3

    フォルダ移動の実践

    「Documentsの中に terminal-practice というフォルダを作り、そこへ移動する」場合の例です。

    BASH
    cd ~/Documents
mkdir terminal-practice
cd terminal-practice
pwd
ls

    それぞれの意味は次のとおりです。

    • cd ~/Documents：Documentsへ移動
    • mkdir terminal-practice：フォルダを作成
    • cd terminal-practice：作ったフォルダへ移動
    • pwd：現在地を確認
    • ls：中身を確認
    4

    パスの読み方

    パスとは、ファイルやフォルダの「住所」です。

    PATH
    /Users/taro/Documents/my-project
    記号意味
    /フォルダの区切り
    .現在のフォルダ
    ..1つ上のフォルダ
    ~自分のホームフォルダ

    例えば、現在のフォルダにある memo.txt は次のようにも表せます。

    PATH
    ./memo.txt
    絶対パスと相対パス

    / で始まるのが「絶対パス」（どこからでも同じ場所を指す）、./ や ../ で始まるのが「相対パス」（今いる場所が基準）です。

    5

    操作を速くするコツ

    5-1. Tabキーで自動補完

    途中まで入力して Tab を押すと、ファイル名やフォルダ名を補完できます。

    入力途中
    cd Doc

    ここで Tab を押すと、候補が1つなら次のようになります。

    補完後
    cd Documents/

    入力ミスを減らせるので、できるだけ使うのがおすすめです。候補が複数あるときは、もう一度 Tab を押すと一覧が表示されます。

    5-2. ↑キーで前のコマンド

    キーボードの ↑ を押すと、直前に実行したコマンドが表示されます。少しだけ内容を変えて再実行するときに便利です。

    5-3. Ctrlキーの便利技

    操作意味
    Ctrl + C実行中の処理を止める（間違えて実行した時や、Claude Code・Codexの処理を途中で止める時にも使う）
    Ctrl + L画面を整理（clear とほぼ同じ）
    Ctrl + A入力行の先頭へ移動
    Ctrl + E入力行の末尾へ移動
    Ctrl + C はコピーではない

    ターミナルでの Ctrl + C は「中断」です。文字のコピーには使えません。macOSでは ⌘ Command + C を使います（詳しくは「11-1. コピペは ⌘C / ⌘V」）。

    5-4. スペースを含む名前

    フォルダ名にスペースがある場合は、引用符で囲みます。

    BASH
    cd "My Documents"

    または、スペースの前に \ を付けます。

    BASH
    cd My\ Documents

    初心者には、引用符で囲む方法がおすすめです。

    6

    コピー・移動・削除

    6-1. コピー・移動・名前変更

    ファイルをコピーします。

    BASH
    cp original.txt copy.txt

    フォルダごとコピーする場合は -r を付けます。

    BASH
    cp -r project project-backup

    名前を変更します。

    BASH
    mv old-name.txt new-name.txt

    別の場所へ移動します。

    BASH
    mv memo.txt ~/Documents

    mv は「移動」と「名前変更」の両方に使われます。

    6-2. 削除は特に注意

    BASH
    rm memo.txt
    削除は元に戻せない

    ターミナルで削除したファイルは、通常ゴミ箱に入りません。元に戻せないことがあります。特に rm -rf フォルダ名 は、意味を完全に理解するまで実行しないでください。

    削除の前は、必ず現在地と対象を確認します。

    BASH
    pwd
ls

    初心者のうちは、Finderやエクスプローラーから削除する方が安全です。

    7

    コマンドの調べ方

    多くのコマンドは、次のようにすると説明を確認できます。

    BASH
    コマンド名 --help

    例：

    BASH
    mkdir --help

    macOSでは、詳しいマニュアルも開けます。

    BASH
    man mkdir

    マニュアル画面を閉じるときは q を押します。

    8

    Codex・Claude Codeの基本形

    まず対象プロジェクトのフォルダへ移動し、その中で起動します。

    BASH
    cd ~/Documents/my-project
ls
claude

    または：

    BASH
    cd ~/Documents/my-project
codex
    起動前に pwd と ls

    「どのフォルダで起動したか」によって、AIが読み書きする対象が変わります。起動前に必ず pwd と ls で現在地と中身を確認すると安全です。

    9

    覚える順番

    まずは、この7つで十分です。

    最初の7つ
    pwd
ls
cd
cd ..
mkdir
clear
Ctrl + C

    慣れてきたら、次を追加します。

    次の5つ
    touch
cat
cp
mv
history
    最も大切な2つの習慣

    rm は便利ですが、最後に慎重に覚えるのがおすすめです。最も大切な習慣は「実行前に現在地を確認する」「分からないコマンドをそのまま貼り付けない」の2つです。

    10

    パイプとリダイレクト

    10-1. パイプ | の考え方

    パイプ | は、左側のコマンドの実行結果を、そのまま右側のコマンドへ渡す仕組みです。

    形
    コマンドA | コマンドB

    イメージは次のとおりです。

    イメージ
    コマンドAの出力 → コマンドBの入力

    1つの | が「パイプ」、複数のコマンドをつないだ全体が「パイプライン」です。

    10-2. まず覚えたい例

    結果の中から文字を検索する

    BASH
    ls | grep "report"

    処理の流れは、ls がフォルダ内の名前を出力 → その結果を grep へ渡す → report を含む行だけ表示、となります。例えば次のファイルがある場合、

    ファイル一覧
    photo.jpg
sales-report.xlsx
staff-report.docx
memo.txt

    結果は次のようになります。

    出力
    sales-report.xlsx
staff-report.docx

    大文字と小文字を区別しない場合は -i を付けます。

    BASH
    ls | grep -i "report"

    長い結果を読みやすくする

    BASH
    ls -la | less

    ls -la の長い結果を、less という閲覧用コマンドに渡します。lessでは次の操作ができます。

    • ↑／↓：上下移動
    • Space：次のページ
    • b：前のページ
    • /文字：検索
    • q：終了

    次のようなコマンドでも便利です。

    BASH
    git diff | less
history | less

    過去に実行したコマンドを検索する

    BASH
    history | grep "claude"

    history | grep "cd " なら、過去に実行したフォルダ移動を探せます。

    履歴にも注意

    コマンドにパスワードやAPIキーを書いていると、それも履歴に残る可能性があります。機密情報をコマンドに直接書くのは避けましょう。

    10-3. 数える・並べ替える

    検索結果の件数を数える

    BASH
    grep -i "error" application.log | wc -l

    結果が 12 なら、error を含む行が12行あるという意味です。wc は word count の略で、-l を付けると行数を数えます。

    ファイル数を数える

    BASH
    find . -type f | wc -l
    • find .：現在のフォルダ以下を探す
    • -type f：ファイルだけを対象にする
    • wc -l：結果の行数を数える

    Markdownファイルだけ数える場合は、名前の条件を加えます。

    BASH
    find . -type f -name "*.md" | wc -l

    *.md は「名前の最後が .md のもの」という意味です。

    容量の大きいフォルダを探す

    BASH
    du -sh * | sort -h

    du -sh * が各ファイル・フォルダの容量を表示し、sort -h が小さい順に並べます。大きい順にするには -hr を使います。

    BASH
    du -sh * | sort -hr

    重複をまとめて件数を数える

    例えば names.txt に次の内容が入っているとします。

    names.txt
    Tanaka
Sato
Tanaka
Suzuki
Sato
Tanaka
    BASH
    sort names.txt | uniq -c | sort -nr

    結果：

    出力
    3 Tanaka
2 Sato
1 Suzuki

    処理は左から順に、sort（同じ名前を隣り合わせにする）→ uniq -c（同じ行の件数を数える）→ sort -nr（件数が多い順に並べる）です。uniq は、同じ行が隣り合っていないとまとめられないため、通常は先に sort します。

    最新の数件だけを見る

    BASH
    history | tail -n 10

    反対に、最初の10行だけなら head を使います。

    BASH
    history | head -n 10

    Gitの履歴でも使えます。

    BASH
    git log --oneline | head -n 10

    10-4. 保存・コピー（tee / pbcopy）

    結果をクリップボードへコピーする

    macOSでは、pbcopy へ渡すとクリップボードにコピーできます。

    BASH
    pwd | pbcopy

    現在のフォルダのパスがコピーされ、そのままメールやチャットへ貼り付けられます。git diff | pbcopy なら、Gitの変更内容をクリップボードへコピーできます。

    画面に表示しながらファイルにも保存する

    通常の > では、結果はファイルに保存され、画面には表示されません。

    BASH
    grep -i "error" application.log > errors.txt

    画面にも表示しながら保存したい場合は、tee を使います。

    BASH
    grep -i "error" application.log | tee errors.txt

    さらに処理を続けることもできます。エラー行を抽出 → 画面に表示しながら errors.txt へ保存 → エラー行数を数える、という流れです。

    BASH
    grep -i "error" application.log | tee errors.txt | wc -l

    10-5. リダイレクトと | の違い

    記号意味
    |次のコマンドへ渡す
    >ファイルへ上書き保存
    >>ファイルの末尾へ追記
    <ファイルを入力として渡す
    • ls | grep "pdf"：ls の結果を grep へ渡す
    • ls > files.txt：ls の結果を files.txt へ保存する
    • ls >> files.txt：既存の files.txt の末尾へ追加する

    | と && は目的が違う

    見た目は似ていますが、役割が違います。| は結果を次へ渡し、&& は成功したら次を実行します。

    BASH
    mkdir new-project && cd new-project

    これは「new-project を作る → 作成に成功した場合だけ、そのフォルダへ移動」という意味です。mkdir new-project | cd new-project としても、mkdir の結果を cd へ渡す意味になり、期待した動作にはなりません。

    初心者はこの5パターンで十分
    • ls | grep "文字"
    • コマンド | less
    • コマンド | wc -l
    • コマンド | sort
    • コマンド | tee result.txt
    パイプの先の削除に注意

    xargs rm など、パイプの先に削除コマンドをつなげる操作もありますが、対象をまとめて誤削除する危険があるため、仕組みに慣れるまでは避けるのが安全です。

    +

    追記：安全に使うための必須知識

    ここからは元記事には無いものの、初心者が事故を防ぐために知っておきたい要点を追記します。

    11-1. コピペは ⌘C / ⌘V

    ターミナルでは Ctrl + C が「中断」なので、文字のコピーには使えません。macOSでは次を使います。

    操作意味
    ⌘ Command + C選択した文字をコピー
    ⌘ Command + V貼り付け

    Windowsのターミナルでは Ctrl + C／Ctrl + V が使えることが多いですが、実行中は「中断」の意味になる場合があります。

    貼り付ける前に中身を読む

    ネットで見つけたコマンドをそのまま貼り付けて実行しないこと。特に rm・sudo を含むものや、意味の分からない長い1行コマンドは、内容を理解してから実行します。

    11-2. 上書き・削除の落とし穴

    cp と mv は、コピー先・移動先に同じ名前のファイルがあると、確認なしで上書きします。大事なファイルを消さないよう、確認を挟むオプション -i を付けると安全です。

    コマンド意味
    cp -i 元 先上書きの前に確認する
    mv -i 元 先上書きの前に確認する
    rm -i ファイル削除の前に1件ずつ確認する

    -i は interactive（確認する）の意味です。

    11-3. sudoと管理者権限

    sudo は「管理者権限で実行する」コマンドです。システム全体を変更できてしまうため影響が大きく、失敗すると元に戻せないことがあります。

    sudo をむやみに使わない

    意味の分からない sudo 付きコマンドは、実行も貼り付けもしないこと。パスワードを求められたからといって安心せず、何をするコマンドかを必ず確認します。

    11-4. 隠しファイルと大文字小文字

    ターミナルは、ファイル名の大文字と小文字を区別します（Memo.txt と memo.txt は別物です）。

    また、.（ドット）で始まるファイルは「隠しファイル」で、通常の ls では表示されません。.git や .env などがこれにあたります。表示するには -a を付けます。

    BASH
    ls -a
    .env などの扱い

    隠しファイルにはAPIキーなど重要情報が入っていることがあります。中身を安易に画面共有・コピーして外部へ貼らないよう注意してください。

    11-5. 困ったときの脱出術

    状況対処
    コマンドが終わらない／間違えたCtrl + C
    プロンプトが > や quote> のまま戻らない（引用符の閉じ忘れ）Ctrl + C でキャンセル
    command not found と出るスペルミスか未インストール。which コマンド名 で在り処を確認
    画面が乱れたclear または Ctrl + L
    現在地が分からなくなったpwd と ls
    Finderで今のフォルダを開きたいopen .（macOS）

    ターミナルそのものを終了するには exit と入力します。
