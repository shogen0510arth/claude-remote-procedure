宿泊施設管理画面でのプラン登録・文章修正・料金／販売期間の入力・写真変更を、担当者がチャットで依頼できる専用AIエージェントを作る、PC初心者向けの手順書。Claude Code＋Playwright＋Claude Agent SDK。

目次

    この手順書の「画面イメージ」について

    本書の「画面イメージ」は操作の考え方を説明するための模式図です。楽天トラベル・一休.comの実際の管理画面は、契約内容や画面更新によって、項目名・配置が異なります。

    1

    この仕組みで実現すること

    最終的には、担当者が次のようにチャットで依頼できる仕組みを作ります。

    依頼例 LOQUAT西伊豆の楽天トラベルと一休で、既存の「スタンダード2食付きプラン」を複製してください。
    販売期間は2026年9月1日から11月30日まで。
    料金は元プランの10％増し。
    添付した秋の料理写真を1枚目にしてください。
    公開はせず、入力内容を私に確認してください。

    AIは以下を順番に行います。

    1. 指示の不足項目を確認する
    2. 対象施設・対象OTAを特定する
    3. 現在の設定内容を読み取る
    4. 変更前の画面を保存する
    5. プランや写真を入力する
    6. 公開直前で停止する
    7. 変更内容を担当者へ報告する
    8. 担当者の承認後に公開する
    9. 公開後の内容を再確認して報告する
    担当者がチャットで依頼
    ↓
    Claudeが指示を整理
    ↓
    PlaywrightがOTA管理画面へ入力
    ↓
    変更内容と画面を提示
    ↓
    承認
    公開・再確認・完了報告
    差し戻し
    指示の整理へ戻る

    絶対に守る安全ルール

    3-1. 公開は必ず人間が承認する

    最初から完全自動公開にはしません。

    入力
    ↓
    内容確認
    ↓
    担当者が承認？
    ↓
    はい
    公開
    いいえ
    修正または中止

    AIに、次の操作を無条件で許可してはいけません。

    • 公開・販売開始
    • プラン削除
    • 写真の全削除
    • 売止め・在庫ゼロ設定
    • 料金の一括変更
    • 既存プランへの上書き

    3-2. パスワードや認証コードをチャットに書かない

    • ログインID、パスワード、ワンタイムパスワードは担当者自身が入力します。
    • Claudeへのチャットにパスワードを貼り付けません。
    • APIキーもClaudeへの通常のプロンプトやプログラム本文に書きません。

    3-3. 予約者情報を見せない

    OTA管理画面には氏名、電話番号、メールアドレス、宿泊履歴などが含まれます。

    • 「プラン管理」「施設情報」「写真管理」だけを対象にします。
    • 「予約一覧」「顧客情報」「メッセージ」画面を開かせません。
    • 可能ならOTA側で設定専用の権限制限アカウントを発行します。

    3-4. 自動操作の可否をOTAに確認する

    楽天トラベル・一休.comの担当者へ、次の文面で確認してください。

    確認文面 当社施設の管理画面について、RPAまたはブラウザ自動操作を用いて、プラン登録や画像更新を行うことは可能でしょうか。
    また、施設情報、プラン、画像を更新できるAPIまたは正式な外部連携手段があればご案内ください。
    公開・確定操作は当社担当者の承認後に行う運用を予定しています。

    APIやサイトコントローラーから変更できる項目はそちらを優先し、ブラウザ操作はAPIで対応できない項目に限定します。

    Google Chromeを準備する

    6-1. Chromeをインストールする

    すでにGoogle Chromeが入っていれば、この作業は不要です。

    1. ブラウザで「Google Chrome ダウンロード」と検索する
    2. Google公式サイトを開く
    3. Chromeをダウンロードを押す
    4. ダウンロードしたファイルを開く
    5. 画面に従ってインストールする

    6-2. OTA専用のChromeプロファイルを作る

    1. Chromeを開く
    2. 右上の丸いプロフィールアイコンを押す
    3. 追加を押す
    4. アカウントなしで続行または業務用Googleアカウントを選ぶ
    5. 名前を「ARTH OTA作業」にする
    6. デスクトップショートカットを作成する
    Chromeのプロフィール選択
    ○ 個人用
    ○ ARTH OTA作業 ← こちらを使用
    ＋ 追加
    注意

    個人用Chromeのプロフィールで作業しないでください。AIが操作するタブに、メール、銀行、契約書、顧客名簿などを開かないでください。

    Node.jsをインストールする

    Playwrightや専用アプリを動かすためにNode.jsを使用します。

    1. Node.js公式ダウンロードページを開く
    2. LTSと書かれた安定版を選ぶ
    3. Windowsなら.msi、Macなら.pkgのインストーラーをダウンロードする
    4. ダウンロードしたファイルを開く
    5. 基本的にはすべて初期設定のままNextまたは続けるを押す
    6. インストール完了後、開いているPowerShellまたはターミナルを一度閉じる

    インストール確認

    WindowsはスタートメニューからPowerShell、Macはターミナルを開き、次を1行ずつ入力します。

    BASH
    node -v
npm -v

    正常な画面イメージ

    TEXT
    PS C:\Users\担当者> node -v
v24.x.x

PS C:\Users\担当者> npm -v
11.x.x

    数字が表示されれば成功です。認識されていませんやcommand not foundと表示された場合は、PCを再起動してもう一度確認します。

    Claude Codeをインストールする

    公式手順はClaude Codeセットアップで確認できます。

    8-1. Windowsの場合

    1. 画面下のスタートボタンを押す
    2. PowerShellと入力する
    3. Windows PowerShellまたはターミナルを開く
    4. 次の1行をコピーして貼り付ける
    5. Enterキーを押す
    POWERSHELL
    irm https://claude.ai/install.ps1 | iex

    別の公式方法として、次も利用できます。

    POWERSHELL
    winget install Anthropic.ClaudeCode

    8-2. Macの場合

    1. commandキーとスペースキーを同時に押す
    2. ターミナルと入力する
    3. ターミナルを開く
    4. 次の1行をコピーして貼り付ける
    5. Returnキーを押す
    BASH
    curl -fsSL https://claude.ai/install.sh | bash

    8-3. インストール確認

    BASH
    claude --version

    バージョン番号が表示されれば成功です。

    Claude Codeを初めて起動する

    OTA-Agentフォルダにいる状態で、次を入力します。

    BASH
    claude

    初回はログインや利用許可の案内が表示されます。画面の指示に従い、Claudeアカウントでログインしてください。

    起動後の画面イメージ

    TEXT
    Claude Code

現在のフォルダ：OTA-Agent

> ここに日本語で指示を入力します

    最初に入れる安全ルールのプロンプト

    以下をそのまま貼り付けてください。

    プロンプト
    このフォルダでは、楽天トラベル、一休.comなどの宿泊施設管理画面を操作する仕組みを作ります。

必ず次のルールを守ってください。
1. 私が明示的に承認するまで、保存、公開、販売開始、削除、売止めを実行しない。
2. 予約一覧、顧客情報、メッセージ、決済情報を開かない。
3. ログインID、パスワード、ワンタイムパスワードを要求しない。
4. 画面が想定と異なる場合、推測で操作せず停止して報告する。
5. 操作の前後で、対象施設、対象OTA、対象プランを確認する。
6. 料金、日付、人数、税区分は入力後に読み直して照合する。
7. 公開または削除につながるボタンを押す前に、必ず私へ確認する。

まず、このルールを理解したことだけを回答してください。ブラウザ操作はまだ開始しないでください。

    Claudeがルールを要約して返答すれば成功です。

    重要

    このプロンプトだけで安全が保証されるわけではありません。試作段階では担当者が画面を見ながら操作し、本番版ではプログラム側でも公開を禁止します。

    11

    PlaywrightをClaude Codeへ追加する

    11-1. Claude Codeを一度終了する

    Claude Codeの入力欄に次を入力します。

    コマンド
    /exit

    11-2. Playwright MCPを追加する

    通常のPowerShellまたはターミナル画面に戻ったら、次を入力します。

    BASH
    claude mcp add playwright npx @playwright/mcp@latest

    Playwright MCPは、ブラウザの移動、クリック、入力、選択、スクリーンショット取得などをClaudeから実行するための機能です。公式の設定方法はPlaywright MCPドキュメントを参照してください。

    11-3. 接続確認

    再びClaude Codeを起動します。

    BASH
    claude

    Claude Code内で次を入力します。

    コマンド
    /mcp

    一覧にplaywrightが表示され、接続済みになっていれば成功です。

    MCP接続一覧状態
    playwright接続済み

    11-4. 一般サイトで安全にテストする

    次のプロンプトを入力します。

    プロンプト
    Playwrightを使って https://example.com をブラウザで開いてください。
ページのタイトルを読み取り、スクリーンショットを1枚保存してください。
クリックや入力はしないでください。

    ブラウザが開き、Claudeがページタイトルを報告すれば接続成功です。

    メモ

    Playwright MCPは通常、操作中のブラウザが見える状態で起動し、プロジェクトごとにログイン状態を保持できます。詳しくはPlaywright MCPのプロファイル設定を参照してください。

    OTAへ手動でログインする

    12-1. ログイン画面だけを開かせる

    最初は「読み取りだけ」を行う

    写真を変更する

    16-1. 写真ファイルを準備する

    写真は、分かりやすい名前に変更しておきます。

    TEXT
    01_秋料理_メイン.jpg
02_秋料理_前菜.jpg
03_客室_三ノ蔵.jpg

    IMG_8374.jpgのような名前のままだと、別の写真を選ぶ事故が起きやすくなります。

    16-2. 写真アップロードのプロンプト

    プロンプト
    新しいプランの写真設定画面を開いてください。

次の3枚をこの順番で登録してください。
1. 01_秋料理_メイン.jpg
2. 02_秋料理_前菜.jpg
3. 03_客室_三ノ蔵.jpg

条件：
- 1枚目をメイン画像にする。
- 既存写真は削除しない。
- 同じ写真を重複登録しない。
- 各写真のファイル名と表示順を登録後に読み直す。
- 保存または公開はしない。
- 写真一覧のスクリーンショットを保存して報告する。

    写真確認のポイント

    • 別施設の写真を選んでいないか
    • 客室タイプと写真が一致しているか
    • 縦横比やトリミングに問題がないか
    • メイン写真の指定が正しいか
    • 料理の季節や内容がプランと一致しているか
    • クレジット表記が必要な写真ではないか

    Claude APIキーを準備する

    メモ

    ClaudeのPro、Max、Teamなどの月額契約と、Claude APIの料金は別です。専用システムでは通常、Claude ConsoleでAPI利用設定が必要です。

    1. Claude Consoleを開く
    2. 業務用アカウントでログインする
    3. Workspaceを作成する。名前はARTH OTA Agentなどにする
    4. API Keysを開く
    5. Create Keyを押す
    6. キー名をota-agent-localなどにする
    7. 表示されたキーを安全なパスワード管理ツールへ保存する

    APIキーの作成場所についてはClaude API利用案内を参照してください。

    APIキーをPCへ登録する（Windows）

    1. スタートメニューで環境変数と検索する
    2. システム環境変数の編集を開く
    3. 環境変数を押す
    4. 上側の「ユーザー環境変数」で新規を押す
    5. 変数名にANTHROPIC_API_KEYを入力する
    6. 変数値にAPIキーを入力する
    7. OKを押してすべて閉じる
    8. PowerShellを一度閉じて開き直す

    APIキーをPCへ登録する（Mac）

    ターミナルで一時的に設定する場合は次の形式です。

    BASH
    export ANTHROPIC_API_KEY="ここにAPIキー"

    恒久設定や本番運用では、エンジニアに依頼してMacのキーチェーンまたは秘密情報管理サービスを利用してください。

    APIキーの取り扱い

    APIキーをチャット、メール、ソースコード、GitHub、スクリーンショットへ貼らないでください。漏洩した場合は、Claude Consoleで直ちに無効化して再発行します。APIキー安全ガイド

    トラブル時の対処

    症状対処
    claudeが見つからないPowerShell／ターミナルを閉じて開き直す。PCを再起動する
    nodeやnpmが見つからないNode.js LTSを再インストールし、PCを再起動する
    Playwrightが接続されないClaude Codeで/mcpを開き、playwrightの状態を確認する
    ブラウザが開かないClaude Codeを終了し、Playwright MCPの追加コマンドを再確認する
    OTAからログアウトしたAIに認証情報を渡さず、担当者が手動で再ログインする
    二段階認証が出た担当者が手動で入力する。自動化しない
    CAPTCHAが出た自動回避せず、担当者が対応する。頻発する場合はOTAへ相談する
    ボタン名や配置が変わった即時停止し、アダプターを修正する
    公開後にエラーが出た再度公開ボタンを押さず、現在の状態を読み取って担当者へ報告する
    間違った施設を開いた何も変更せずログアウトまたはタブを閉じる
    APIキーを漏らしたClaude Consoleで該当キーを無効化し、新しいキーを発行する
    緊急停止

    アプリやClaude Codeの動作を止めたいときは、PowerShellまたはターミナルでCtrl＋Cを押します。ブラウザ操作中なら、ブラウザを閉じるだけでなく、ターミナル側の処理も停止してください。

    本番運用へ移るための合格基準

    次をすべて満たすまで、本番の自動公開を有効にしません。

    • OTAから自動操作の可否について回答を得た
    • 専用PCまたは専用PCユーザーを用意した
    • 権限を絞ったOTAアカウントを用意した
    • 予約者情報ページへのアクセスを禁止した
    • 読み取り操作を10回連続で成功した
    • 下書き入力を10回連続で成功した
    • 料金・期間・写真順の検算に成功した
    • 画面変更時に推測で操作せず停止できた
    • 公開承認なしでは公開できないことをテストした
    • 同じ承認トークンを2回使えないことをテストした
    • エラー時に自動再公開しないことをテストした
    • 操作ログと変更前後の画像が保存された
    • 担当者が緊急停止方法を理解した
    • エンジニアまたはセキュリティ担当者が本番設定を確認した
